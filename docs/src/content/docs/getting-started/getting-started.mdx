title: Getting Started with Diesel
---

Diesel is a simple and lightweight HTTP server library for Bun.js that provides you with complete control over your API routes and middleware. It is designed to be intuitive and efficient, allowing you to quickly set up a server, define routes, and optimize important paths for faster response times.

With built-in support for TypeScript, DieselJS ensures type safety and improved developer experience, making it easier to catch errors during development. Whether you are building a small application or a large API, DieselJS helps you manage your routes and middleware seamlessly.
## Quick Start

You can install Diesel from npm:


```bash
npm install diesel-core
```
```bash
bun add diesel-core
```
```bash
yarn add diesel-core
```
```bash
pnpm install diesel-core
```

## DieselOptions

You can pass options to the `Diesel` constructor to configure its behaviour:

```typescript
const app = new Diesel({
  logger: true,             // enable built-in request/response logger
  platform: "bun",          // "bun" (default) | "cf" (Cloudflare Workers)
  errorFormat: "json",      // "json" (default) | "text"
  jwtSecret: "my-secret",   // secret used by authenticateJwt / authenticateJwtDB
  idleTimeOut: 10,          // Bun server idle timeout in seconds
  baseApiUrl: "/api/v1",    // prefix added to all file-based routes
  prefixApiUrl: "",         // prefix added to all routes
  enableFileRouting: false, // enable file-based routing (src/routes/)
  pipelineArchitecture: false, // enable new pipeline architecture (experimental)
  onError: false,           // log errors automatically with console.error
  router: "t2",             // router implementation to use
});
```

## HTML files serving (Bun native static routing)

Serve HTML files directly via Bun's native static routing using `app.staticHtml()`:

```typescript
import homePage from "./templates/index.html";
import aboutPage from "./templates/about.html";

app.staticHtml({
  "/" : homePage,
  "/about" : aboutPage,
});
```

**This is the fastest way to serve HTML — Bun handles it natively without going through the Diesel request pipeline.**

## Serve a static files directory

Use `app.static()` to serve files from a directory on the filesystem:

```typescript
// Serve files from ./public under any path
app.static(import.meta.dir + "/public");

// Serve files from ./public only under /assets/*
app.static(import.meta.dir + "/public", "/assets");
```

## Single Page Application (SPA) support

```typescript
app.static(import.meta.dir + "/public");

// Catch-all: return the SPA entry point for client-side routing
app.get("*", (ctx) => {
  return ctx.file(import.meta.dir + "/public/index.html");
});
```
