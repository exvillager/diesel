## Middleware Overview

Middleware functions in Diesel.js allow you to run code before your request reaches the endpoint. There is no `next()` — if everything is fine, simply don't return anything and Diesel will proceed to the next handler. To stop the request (e.g., unauthorized), return a `Response`.

## JWT Authentication Middleware

### Implementation

```typescript
import jwt from 'jsonwebtoken';
import type { ContextType } from 'diesel-core';

const secret = 'your_secret_key';

async function authJwt(ctx: ContextType): Promise<void | Response> {
  try {
    const token = ctx.cookies.accessToken;

    if (!token) {
      return ctx.json({ message: "Authentication token missing" }, 401);
    }

    const user = jwt.verify(token, secret);
    ctx.set('user', user); // Store user in context for downstream handlers
  } catch (error) {
    return ctx.json({ message: "Invalid token" }, 403);
  }
}

// Apply globally to all routes
app.use(authJwt);

// Apply only to a specific path
app.use("/user", authJwt);
```

## Using Multiple Middleware

```typescript
app.use(authJwt, rateLimiter); // global, chained

app.use("/user", authJwt, rateLimiter); // path-specific, chained

app.use(["/user", "/home"], [authJwt, rateLimiter]); // multiple paths, multiple middleware
```

## Inline Route Middleware

You can attach middleware directly to a route handler:

```typescript
app.get("/user/profile", authJwt, (ctx: ContextType) => {
  const user = ctx.get("user");
  return ctx.json({ profile: user });
});

app.post("/post", authJwt, requestLogger, (ctx: ContextType) => {
  return ctx.json({ message: "Post created" });
});
```

All middleware in the chain runs in order. The main handler only runs if none of the middleware returns a `Response`.
