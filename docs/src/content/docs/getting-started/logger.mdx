### Enhancing Request Handling with Loggers in DieselJS

DieselJS ships with two built-in loggers that track the request-response cycle, making debugging and performance analysis straightforward.

## What are loggers?

Loggers in DieselJS log information about incoming requests and outgoing responses, including request methods, URLs, response statuses, and execution times. They work by registering `onRequest`, `onSend`, and `onError` hooks internally.

## Types of Loggers

1. `logger` - Simple, compact request/response logging
2. `advancedLogger` - Detailed JSON-formatted logging with headers and timestamps

---

## Quick Enable via `DieselOptions`

The simplest way to enable the basic logger is to pass `logger: true` when creating the app:

```typescript
import { Diesel } from "diesel-core";

const app = new Diesel({ logger: true });

app.get("/", (ctx) => ctx.text("Hello!"));

app.listen(3000);
```

---

## `logger` — Basic Logging

Import and apply manually for more control:

```typescript
import { Diesel } from "diesel-core";
import { logger } from "diesel-core/logger";

const app = new Diesel();

app.useLogger({ app });
// or equivalently:
// logger({ app });

app.listen(3000);
```

Sample output:
```
<-- GET /
--> GET / 200 5ms
```

You can also pass custom callbacks:

```typescript
app.useLogger({
  app,
  onRequest: (req, pathname) => {
    console.log(`[custom] ${req.method} ${pathname}`);
  },
  onError: (error, pathname, req) => {
    console.error(`[custom error] ${req.method} ${pathname}: ${error.message}`);
  },
});
```

---

## `advancedLogger` — Detailed Logging

```typescript
import { Diesel } from "diesel-core";
import { advancedLogger } from "diesel-core/logger";

const app = new Diesel();

app.useAdvancedLogger({ app });
// or equivalently:
// advancedLogger({ app });

app.listen(3000);
```

Sample output:
```
[INFO] Incoming Request - GET
{
  "timestamp": "2025-03-20T11:00:15.744Z",
  "method": "GET",
  "url": "/",
  "headers": {
    "user-agent": "Mozilla/5.0 ...",
    "content-type": null
  }
}

[INFO] Response Sent - GET
{
  "timestamp": "2025-03-20T11:00:15.753Z",
  "method": "GET",
  "url": "http://localhost:3000/",
  "status": 200,
  "duration": "10ms"
}
```

You can also pass custom callbacks:

```typescript
app.useAdvancedLogger({
  app,
  onRequest: (ctx) => {
    console.log(`[custom] ${ctx.req.method} ${ctx.path}`);
  },
  onSend: (ctx) => {
    // optionally return a Response to override
  },
  onError: (error, path, req) => {
    console.error(`[error] ${req.method} ${path}: ${error.message}`);
  },
});
```

### Conclusion

- Use `new Diesel({ logger: true })` for zero-config basic logging.
- Use `app.useLogger({ app })` or `app.useAdvancedLogger({ app })` for more control.
- `advancedLogger` is best for production debugging with detailed JSON output.
