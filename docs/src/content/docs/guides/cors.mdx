# CORS Configuration

Cross-Origin Resource Sharing (CORS) is a mechanism that allows restricted resources on a web page to be requested from another domain. Diesel ships with a built-in `cors` middleware.

## Import

```typescript
import { cors } from "diesel-core/cors";
```

## Basic Usage

```typescript
import { Diesel } from "diesel-core";
import { cors } from "diesel-core/cors";

const app = new Diesel();

app.use(cors({
  origin: "*", // allow all origins
}));

app.get("/api/data", (ctx) => {
  return ctx.json({ message: "CORS-enabled response" });
});

app.listen(3000);
```

## Configuration Options

| Option | Type | Default | Description |
|---|---|---|---|
| `origin` | `string \| string[]` | — | Allowed origin(s). Use `"*"` for all. |
| `methods` | `string[]` | `["GET","POST","PUT","DELETE","OPTIONS"]` | Allowed HTTP methods. |
| `allowedHeaders` | `string[]` | `["Content-Type","Authorization"]` | Allowed request headers. |
| `exposedHeaders` | `string[]` | `[]` | Headers exposed to the browser. |
| `credentials` | `boolean` | `false` | Whether to allow cookies/credentials. |
| `maxAge` | `number` | — | Preflight cache duration in seconds. |

## Example: Restrict to specific origins

```typescript
app.use(cors({
  origin: ["http://localhost:5173", "https://myapp.com"],
  methods: ["GET", "POST", "PUT", "DELETE"],
  allowedHeaders: ["Content-Type", "Authorization"],
  credentials: true,
}));
```

## Example: Allow a single origin

```typescript
app.use(cors({
  origin: "https://myapp.com",
  credentials: true,
}));
```

## How it works

- If the request `Origin` matches the configured allowed origins, Diesel sets `Access-Control-Allow-Origin` accordingly.
- For `OPTIONS` preflight requests, Diesel responds with `204` and sets `Access-Control-Max-Age` if `maxAge` is configured.
- If the origin is not allowed, Diesel returns `403 CORS not allowed`.
