In this document, we will cover the HTTP methods used in routing (GET, POST, etc.) and how to implement sub-routing in your application.

## HTTP Methods

Diesel supports all standard HTTP methods:

```typescript
app.get('/home', (ctx: ContextType) => {
  return ctx.text('Welcome to the home page!');
})

app.post("/upload", (ctx: ContextType) => {
    // your logic here
})

app.put("/update", (ctx: ContextType) => {
    // your logic here
})

app.patch("/patch", (ctx: ContextType) => {
    // your logic here
})

app.delete("/delete-user", (ctx: ContextType) => {
    // your logic here
})

app.head("/check", (ctx: ContextType) => {
    // your logic here
})

app.options("/options", (ctx: ContextType) => {
    // your logic here
})

// matches any HTTP method
app.any("/catch-all", (ctx: ContextType) => {
    return ctx.json({ method: ctx.req.method });
})
```

### Register a route for multiple methods at once

Use `app.onMethod()` to handle one or more HTTP methods on a single path:

```typescript
app.onMethod(["GET", "POST"], "/data", (ctx: ContextType) => {
  return ctx.json({ method: ctx.req.method });
});
```

### Route-level redirect

```typescript
// Redirects all requests to /old-path → /new-path (preserves params and query)
app.redirect("/old-path", "/new-path");
```

## Sub-Routing in Diesel

Diesel offers two approaches for sub-routing: `app.route()` (merges routes) and `app.sub()` (recommended, keeps routers isolated).

### Using `app.sub()` (Recommended)

`app.sub()` is the recommended way to compose sub-routers. It keeps the child router's middleware isolated to its own routes.

import {Code} from '@astrojs/starlight/components';
export const newDieselCode = `
import { Diesel } from "diesel-core";

export const userRoute = new Diesel();

userRoute.get("/register/:id", (ctx: ContextType) => {
  return ctx.text('User registered with ID: ' + ctx.params.id);
});

userRoute.get("/login", (ctx: ContextType) => {
  return ctx.text("Login route");
});
`
export const fileName = `user.route.ts`;
export const highlights = ['file', 'CMS'];

<Code code={newDieselCode} lang='ts' title={fileName} mark={highlights}/>

**Mounting the sub-router in the main app:**

```typescript
import { userRoute } from './user.route';

// All userRoute routes will be prefixed with /api/v1/user
app.sub("/api/v1/user", userRoute);
```

### Using `app.route()`

`app.route()` merges the child router's routes into the parent. Useful when you want the child's middleware to also apply at the parent level.

```typescript
import { userRoute } from './user.route';

app.route("/api/v1/user", userRoute);
```

**`app.route()` takes two arguments:**
1. **path**: The base path prefix (e.g., `/api/v1/user`).
2. **Router instance**: The Diesel instance containing the routes to register.

### Mounting 3rd-party handlers with `app.mount()`

`app.mount()` lets you integrate any fetch-compatible handler (e.g., another Diesel app or a Hono app) under a path prefix:

```typescript
import { Diesel } from "diesel-core";
import anotherApp from "./another-app";

// All requests to /legacy/* are forwarded to anotherApp
app.mount("/legacy", anotherApp);
```

### Conclusion

**By understanding and utilizing HTTP methods along with sub-routing, you can create a well-structured and efficient routing system. Use `app.sub()` for isolated child routers, `app.route()` for merged routers, and `app.mount()` for integrating 3rd-party handlers.**
