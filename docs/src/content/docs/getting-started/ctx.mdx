title: Understanding `ctx` Context
---

## Overview
The `ctx` context is an essential object that provides various methods and properties for handling requests and responses in your application. It simplifies request handling, response formatting, and state management within the server.

## Properties

- **req**: The incoming `Request` object.
- **server**: The Bun `Server` instance (undefined in Cloudflare Workers).
- **path**: The request pathname (e.g. `/api/users`).
- **url**: Lazily parsed `URL` object for the request.
- **env**: Environment bindings (available in Cloudflare Workers).
- **executionContext**: Cloudflare Workers execution context.
- **headers**: Custom response headers accumulated via `setHeader()`.
- **query**: Parsed query parameters (lazily computed).
- **params**: Extracted dynamic route parameters.
- **cookies**: Parsed cookies from the request (lazily computed).
- **body**: The parsed request body (lazily computed Promise).
- **ip**: The client's IP address.

## Methods

### Request Handling
- **get ip**: Retrieves the client's IP address. Uses Bun's `server.requestIP()` or the `CF-Connecting-IP` header on Cloudflare.
  ```typescript
  const clientIP = ctx.ip;
  ```
- **get query**: Retrieves parsed query parameters as a plain object.
  ```typescript
  const queryParams = ctx.query;
  const name = ctx.query.name;
  ```
- **get params**: Retrieves dynamic route parameters.
  ```typescript
  const id = ctx.params.id;
  ```
- **get body**: Retrieves the parsed request body. Supports JSON, URL-encoded, and multipart form data.
  ```typescript
  const body = await ctx.body;
  ```

### Response Handling
- **setHeader(key, value)**: Sets a custom response header. Returns `this` for chaining.
  ```typescript
  ctx.setHeader("X-Custom-Header", "value");
  ```
- **removeHeader(key)**: Removes a previously set response header.
  ```typescript
  ctx.removeHeader("Cache-Control");
  ```
- **text(data, status?, customHeaders?)**: Sends a plain text response.
  ```typescript
  return ctx.text("Hello, World!", 200);
  return ctx.text("Not found", 404, { "X-Info": "missing" });
  ```
- **json(data, status?, customHeaders?)**: Sends a JSON response.
  ```typescript
  return ctx.json({ message: "Success" }, 200);
  return ctx.json({ error: "Not found" }, 404, { "X-Info": "missing" });
  ```
- **send(data, status?, customHeaders?)**: Sends a response with an automatically detected MIME type (JSON, text, or binary).
  ```typescript
  return ctx.send({ key: "value" }, 200);
  return ctx.send("plain text");
  return ctx.send(uint8Array, 200);
  ```
- **file(filePath, mimeType?, status?, customHeaders?)**: Serves a file as a response.
  ```typescript
  return ctx.file("./downloads/sample.pdf");
  return ctx.file("./image.png", "image/png", 200);
  ```
- **redirect(path, status?)**: Redirects to a specified path. Default status is `302`.
  ```typescript
  return ctx.redirect("/login");
  return ctx.redirect("/new-page", 301);
  ```
- **stream(callback)**: Sends a streaming response via `ReadableStream`.
  ```typescript
  return ctx.stream((controller) => {
    controller.enqueue(new TextEncoder().encode("chunk 1"));
    controller.enqueue(new TextEncoder().encode("chunk 2"));
  });
  ```

### Context Management
- **set(key, value)**: Stores a typed value in the request-scoped context.
  ```typescript
  ctx.set("user", { id: 1, name: "Alice" });
  ```
- **get(key)**: Retrieves a value from the request-scoped context.
  ```typescript
  const user = ctx.get<{ id: number; name: string }>("user");
  ```

### Cookie Handling
- **setCookie(name, value, options?)**: Appends a `Set-Cookie` header. Returns `this` for chaining.
  ```typescript
  ctx.setCookie("session", "abc123", { httpOnly: true, secure: true, sameSite: "Strict" });
  ```
- **get cookies**: Retrieves all cookies from the request as a plain object.
  ```typescript
  const token = ctx.cookies.accessToken;
  ```

## Example Usage

```typescript
app.get("/profile", async (ctx) => {
  const user = ctx.get("user");
  if (!user) {
    return ctx.json({ error: "Unauthorized" }, 401);
  }
  return ctx.json({ profile: user });
});
```

This documentation provides a structured and clear explanation of `ctx` functionalities to help developers effectively utilize it in their applications.
